# RTK Codex Workspace

- This repo now uses a TypeScript CLI, not shell installer scripts.
- Build locally with `npm install && npm run build`.
- Install the machine-level shims and shared Codex override with `node /Users/rolandk/Developer/rtk-codex/dist/cli.js install`.
- Remove only the managed override block with `node /Users/rolandk/Developer/rtk-codex/dist/cli.js uninstall`.
- Refresh upstream-supported shim commands with `node /Users/rolandk/Developer/rtk-codex/dist/cli.js update`.
- Installed shims live under `~/.rtk-codex` by default so the local checkout path does not affect Codex after installation.
- Repo-local generated artifacts are not source of truth. `dist/*`, `shims/bin/*`, `shims/shim-commands.txt`, and `shims/shim-commands.upstream.txt` are regenerated by the CLI when needed.
- Codex CLI and Codex Desktop pick up the shim environment from `~/.codex/config.toml` via `shell_environment_policy`.
- When the shim environment is active, use normal commands such as `git status`, `ls`, `cat AGENTS.md`, and `gh pr view`. The wrappers will route selected commands through `rtk`.
- Keep shell usage ordinary. Do not force explicit `rtk ...` prefixes unless you need a specific RTK subcommand that is not covered by the wrappers.
- Wrapper scope is intentionally narrow. If a command needs exact native behavior, call the real binary directly with an absolute path such as `/opt/homebrew/bin/git`.
- Validation logs default to `/tmp/codex-rtk-shim.log` unless `RTK_SHIM_LOG_FILE` is overridden. Repo-local scratch logs live under `/Users/rolandk/Developer/rtk-codex/shims/logs/` when they are generated.
